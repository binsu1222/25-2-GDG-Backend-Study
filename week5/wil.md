## 객체지향
Java는 객체 지향 언어이고, Spring은 Java 기반의 프레임워크이다.  
즉, 스프링은 좋은 객체 지향 프로그램을 개발할 수 있도록 도움을 주는 프레임워크인 셈이다.

### 특징
1. 추상화
2. 캡슐화
3. 상속
4. 다형성

위 특징 중 다형성은 객체지향 언어의 중요한 특징이다.     

객체를 사용하는 클라이언트는 대상의 역할만 알 뿐, 실제 구현을 몰라도 된다.  
다형성을 통해 구현 대상 자체가 변경되더라도 클라이언트는 영향을 받지 않는다.

> 즉, 다형성은 프로그램을 유연하고, 변경에 용이하게 만들어준다 !

위의 사항을 코드에 구체적으로 적용해보면    
- 역할 -> 인터페이스
- 구현 -> 인터페이스를 구현한 클래스    
로 볼 수 있다.

## SOLID 원칙
- SRP : 단일 책임 원칙
- OCP : 개방-폐쇄 원칙
- LSP : 리스코프 치환 원칙
- ISP : 인터페이스 분리 원칙
- DIP : 의존관계 역전 원칙
