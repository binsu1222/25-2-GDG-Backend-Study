## 데이터 베이스 접근하기
서버는 DB가 이해할 수 있는 SQL을 연결된 커넥션을 통해 DB에 전달 => 전달된 SQL을 DB가 수행하고 그 결과를 서버가 가져간다.

Java에서 DB에 접근하기 위판 표준 API가 JDBC이다.


    하지만 SQL을 직접 다루는 방식은 SQL에 의존적이고, 객체 - DB 패러다임이 불일치 하는 문제가 발생한다. 그래서 나온 개념이 ORM이다.

### ORM
Object-Relational-Mapping 의 약자로, 말그대로 객체와 관계형 데이터베이스의 데이터를 맵핑해주는 기술이다.

스프링에서는 ORM 기술 표준인 JPA를 사용하고, JPA도 사실상 인터페이스이기 때문에 실제 구현체로 Hibernate를 사용한다.

### JPA
- JPA 기능의 대부분은 엔티티 매니저가 제공
- 엔티티 메니저를 사용해서 SQL 직접 작성 없이 엔티티 등록/수정/조회/삭제 가능

### 영속성 컨텍스트
엔티티 생명 주기

    <비영속, 영속, 준영속, 삭제>

- 1차 캐시
- 동일성 보장
- 트랜젝션을 지원하는 쓰기 지연
- 변경 감지
